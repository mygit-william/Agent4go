# 安全策略

## 报告安全问题

如果你发现了 Nanobot-Go 中的安全问题，请通过以下方式联系我们：

- **邮箱**: security@nanobot-go.com
- **GitHub Security Advisory**: [GitHub Security](https://github.com/mygit-william/nanobot-go/security/advisories/new)

**请不要在公共 Issues 中报告安全问题！**

## 安全更新

我们会及时处理和修复安全问题。重要安全更新将通过以下渠道通知：

- GitHub Security Advisories
- 项目发布说明
- 邮件列表（如适用）

## 安全最佳实践

### 部署建议

#### 1. API Key 管理
- **不要硬编码 API Key**
- 使用环境变量或外部配置文件
- 定期轮换密钥

```bash
# 使用环境变量
export ZHIPU_API_KEY="your-api-key"
./nanobot -mode cli
```

#### 2. 网络隔离
- 在生产环境中运行在防火墙后
- 限制不必要的网络访问
- 使用 VPN 或私有网络连接 LLM 服务

#### 3. 权限最小化
- 使用 `permissions.mode: "default"` 进行生产部署
- 避免授予不必要的执行权限
- 定期审查权限设置

#### 4. 日志监控
- 启用详细的日志记录
- 监控异常活动模式
- 设置日志轮转和归档

### 运行时安全

#### 输入验证
- 所有用户输入都经过验证
- 防止命令注入攻击
- 限制文件大小和类型

#### 资源限制
- 设置内存使用上限
- 限制并发任务数量
- 监控 CPU 使用率

```bash
# 检查依赖安全性
go list -m all | grep github.com
go get -u ./...

# 检查代码质量
gofmt -s -w .
golint ./...
```

## 应急响应

### 发现漏洞后的步骤
1. **立即报告**: 联系安全团队
2. **初步评估**: 确定严重程度和影响范围
3. **修复开发**: 快速开发安全补丁
4. **测试验证**: 确保修复有效且不引入新问题
5. **发布更新**: 发布安全更新版本
6. **通知用户**: 告知受影响的用户

### 紧急联系方式
- **主要联系人**: security@nanobot-go.com
- **响应时间**: 24小时内回复
- **严重性分类**:
  - 高危: 24小时内处理
  - 中危: 72小时内处理
  - 低危: 1周内处理
